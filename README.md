# Проект: ИИ-ассистент для сети фитнес-клубов

## 📋 О проекте
**ИИ-ассистент для сети фитнес-клубов** — это интеллектуальное решение, предназначенное для автоматизации коммуникации с клиентами, разгрузки персонала и оптимизации процесса записи на тренировки. Ассистент работает в режиме 24/7, обеспечивая мгновенные, точные и персонализированные ответы, а также бесшовно интегрируется с рабочими календарями тренеров и администраторов.

---

## 🎯 Цели и задачи проекта
1. **Автоматизация рутины**: Снижение нагрузки на администраторов и колл-центр за счет делегирования типовых вопросов ИИ.
2. **Повышение конверсии**: Мгновенная реакция на запросы клиентов в любое время суток, что минимизирует потерю лидов.
3. **Исключение человеческого фактора**: Предотвращение двойных записей и ошибок при планировании расписания.
4. **Улучшение клиентского опыта**: Предоставление удобного, естественного и полезного интерфейса общения для клиентов.

---

## ⚙️ Ключевой функционал

### 1. Осмысленное ведение диалога
- Использование продвинутых NLP-алгоритмов для понимания контекста, намерений (intents) и тональности сообщений клиента.
- Поддержка естественного, человекоподобного стиля общения без шаблонных «роботизированных» фраз.
- Удержание контекста беседы для корректной обработки уточняющих вопросов.

### 2. Интеллектуальные ответы на основе базы знаний (RAG)
- Мгновенные ответы на частые вопросы (FAQ): стоимость абонементов, расписание групповых занятий, квалификация тренеров, правила клуба, наличие парковки/душевых и т.д.
- Динамическое обновление базы знаний без необходимости перепрограммирования ассистента.

### 3. Автоматическая запись и синхронизация с Google Calendar
- Интерактивный подбор свободного времени для персональной или групповой тренировки.
- Автоматическая проверка доступности слотов в Google Calendar конкретного тренера или зала.
- Создание события в календаре с указанием имени клиента, типа тренировки и контактных данных.
- Отправка клиенту автоматического подтверждения с деталями записи (время, место, имя тренера).

## 🔄 Архитектура рабочего процесса (Workflow)

1. **Инициация**: Клиент пишет сообщение в мессенджер или чат на сайте.
2. **Обработка**: Savvy принимает сообщение и передает его в Qwen с учетом системного промпта и актуальной базы знаний.
3. **Анализ намерения**: 
   - *Если это вопрос*: Qwen генерирует точный ответ на основе базы знаний, Savvy отправляет его клиенту.
   - *Если это запрос на запись*: Ассистент уточняет тип тренировки и предпочтительное время.
4. **Интеграция**: Savvy через API запрашивает свободные слоты в Google Calendar.
5. **Подтверждение**: Ассистент предлагает клиенту доступные варианты. После выбора клиента, Savvy создает событие в Google Calendar и отправляет итоговое подтверждение.

## 💼 Бизнес-ценность и ожидаемые результаты

- **Экономия времени**: До 70% рутинных запросов обрабатываются без участия человека.
- **Масштабируемость**: Возможность подключения новых клубов сети к единой системе за считанные часы путем простого обновления базы знаний.
- **Прозрачность**: Вся история переписок и записей сохраняется и доступна для анализа руководством.
- **Снижение операционных расходов**: Оптимизация штата администраторов или перераспределение их времени на более ценные задачи (продажи, сервис в зале).
